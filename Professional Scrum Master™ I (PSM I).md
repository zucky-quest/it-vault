**【目標時期：1〜2ヶ月以内】** 自社開発の現場で「スクラムの専門家」として認知されるための第一歩です。

### **1. 学習フェーズ（2週間〜1ヶ月）**

- **スクラムガイドの徹底読解:**

  - [最新のスクラムガイド（日本語版）](https://scrumguides.org/docs/scrumguide/v2020/2020-Scrum-Guide-Japanese.pdf)を、一字一句漏らさず3回読みます。特に「各ロールの責務」と「作成物の確約（Commitment）」の対応関係を暗記してください。

- **無料模試（Scrum.org Open Assessments）:**

  - [Scrum Open](https://www.scrum.org/open-assessments/scrum-open)を毎日1回解く（**合格基準:** 100%・5分以内・3日連続）。

  - [Product Owner Open](https://www.scrum.org/open-assessments/product-owner-open)（15問）— Scrum Open が安定したら実施。**合格基準:** 100%。

  - [Scrum Developer Open](https://www.scrum.org/open-assessments/scrum-developer-open)（30問）— PO Open が安定したら実施。**英語のみ**（日本語版なし）。翻訳は下記「英語問題の読み方」参照。**合格基準:** 85%以上（できれば100%）。1回受けて85%超なら次へ進んでよい（毎日ループ不要）。

  - [Open Assessments 一覧](https://www.scrum.org/open-assessments) の**すべて**を解く必要はない（Nexus / Kanban / AI / EBM 等は PSM I 単独ではスキップ可）。

  - **Open の言語:** [Scrum Open（日本語）](https://www.scrum.org/open-assessments/scrum-open-jp)・[PO Open（日本語）](https://www.scrum.org/open-assessments/product-owner-open-jp) は日本語可。**Developer Open は英語のみ**。

- **Udemyでの問題演習:**

  - 「PSM I Practice Exams」等のコースを購入（セール時なら2,000円弱）。英語問題は **DeepL で部分翻訳**（下記参照）し、「なぜその選択肢が正解なのか」の根拠をスクラムガイドから探す練習をします。

### **3. 本番試験（PSM I）の概要**

[公式ページ](https://www.scrum.org/assessments/professional-scrum-master-i-certification) / [Suggested Reading](https://www.scrum.org/resources/suggested-reading-professional-scrum-master)

#### **試験仕様**

| 項目 | 内容 |
| ---- | ---- |
| 受験料 | **$200 USD**（1回分。再受験も同額） |
| 問題数 | **80問** |
| 制限時間 | **60分**（1問あたり約45秒） |
| 合格ライン | **85%（68問以上正解）** |
| 形式 | 単一選択・複数選択・True/False |
| 受験方法 | オンライン（自宅等）。**オープンブック**だが時間が厳しい |
| 言語 | 英語・**日本語**・簡体中国語 |
| 有効期限 | **生涯有効**（更新不要） |
| パスワード | 購入後いつでも受験可。ただし**1パスワード＝1回のみ** |

#### **出題範囲（Focus Areas）**

Scrum Guide を主ソースとし、シナリオ問題で「最も適切な答え」「この状況で次に取るべき行動は？」を問う形式が多い。

1. **Understanding and Applying the Scrum Framework**（中核）
   - 経験主義（Transparency / Inspection / Adaptation）
   - Scrum Values（Commitment, Focus, Openness, Respect, Courage）
   - Scrum Team（SM / PO / Developers の Accountabilities）
   - Events（Sprint, Planning, Daily Scrum, Review, Retrospective）
   - Artifacts & Commitments（Product Goal, Sprint Goal, Definition of Done）
2. **Developing People and Teams**
   - 自己管理チーム、ファシリテーション、コーチング
3. **Managing Products with Agility**
   - Product Backlog 管理、価値最大化、ステークホルダー、リリース計画

#### **本番と Scrum Open の違い**

| 観点 | Scrum Open | PSM I 本番 |
| ---- | ---------- | ---------- |
| 問題数 | 約30問 | **80問** |
| 時間 | 60分（余裕あり） | **60分（厳しい）** |
| 問題プール | 限定的（同じ問題が出る） | ランダム抽選・未見問題多い |
| 出題形式 | 比較的ストレート | **シナリオ・「次に取るべき行動」型が多い** |
| 難易度 | 本番より易しい | Open 100% でも落ちる例あり |

#### **英語問題の読み方（翻訳のコツ）**

Chrome 等の**ページ全体翻訳**は、選択肢のレイアウトが崩れたり、Scrum 用語が誤訳されたりして試験向きではない。**DeepL で必要な部分だけ翻訳**するのが現実的。

**基本方針**

| 場面 | 方針 |
| --- | --- |
| Scrum Open / PO Open | **[日本語版](https://www.scrum.org/open-assessments/scrum-open-jp)を使う**（翻訳不要） |
| **PSM I 本番** | **日本語で受験**（2025年4月〜。翻訳不要） |
| Developer Open / Real Mode / Volkerdon 等 | 英語のみ → **DeepL で部分翻訳** |

**DeepL の使い方（試験中）**

1. **ページ全体は翻訳しない** — 問題文だけコピーして DeepL（[Web版](https://www.deepl.com/ja/translator) またはデスクトップアプリ）に貼る
2. **長いシナリオは2回に分ける** — ①状況説明 → ②設問。選択肢は A/B/C/D を**1つずつ**翻訳（まとめて貼ると崩れやすい）
3. **DeepL を横に置く** — 試験タブと DeepL を左右に並べ、コピペで行き来（ブラウザ拡張の自動翻訳はオフ）
4. **怪しい訳は日本語 Scrum Guide で確認** — 例: "accountability" → 「説明責任」、「commitment" → 「確約」
5. **学習時は英日 Scrum Guide を並べて読む** — 頻出語句が頭に入ると、翻訳回数が減る

**DeepL より向かないもの**

- Chrome / Edge の**自動ページ翻訳** — UI が崩れ、複数選択肢の文脈が壊れやすい
- Google 翻訳のページ全体翻訳 — 同上

**時間がかかるのは正常** — Developer Open や Real Mode は英語前提。本番は日本語受験に切り替えれば、翻訳時間はほぼゼロになる。

#### **受験のコツ**

- 集中できる**土日午前**など、60分確保できる時間帯に受験
- **本番は日本語受験**を選ぶ（翻訳ツール不要）
- Scrum Guide は別タブで開けるが、**毎問参照は時間切れ**になる
- 明らかに Scrum Guide に反する選択肢を先に除外する
- 複数「正しそう」な場合は **「いまこの場面で最初に取るべき一手」** を選ぶ

#### **受験準備の目安**

| 条件 | 判断 |
| ---- | ---- |
| Scrum Open 100% + 5分以内が **3日連続** | 受験 OK |
| Product Owner Open も 100% | より安全 |
| Scrum Developer Open **85%以上**（できれば100%） | 公式推奨の追加模試クリア |
| Volkerdon / Mikhail Real Mode で **90%以上** | 本番向き |
| ミスした問題の理由を Scrum Guide の一文で説明できる | 弱点解消済み |

#### **受験までの自分の Next Action（学習者側）**

スクラム用語の「next action」とは別に、受験準備としての次の一手の目安。

1. **いま:** Scrum Open・PO Open を維持（100%・5分以内）。並行または次に [Scrum Developer Open](https://www.scrum.org/open-assessments/scrum-developer-open)（30問）を受験
2. **Developer Open が85%以上**（できれば100%）→ Scrum Open が**3日連続100%**か最終確認
3. **上記3つの Open が安定** → **[Mikhail Lapshin Real Mode](https://mlapshin.com/index.php/scrum-quizzes/sm-real-mode/)**（80問・60分）で本番形式に慣れる。90%以上なら本番へ
4. **Real Mode も安定** → 本番パスワード購入（約3.2万円）・受験日を土日午前で確保（**本番は日本語受験可**）
5. **余力があれば:** [Volkerdon](https://www.volkerdon.com/) で追加演習
6. **受験当日:** 60分確保・Scrum Guide を別タブで開ける状態にしておく

> たまに1問ミスする段階なら、**パスワード購入はまだ早い**。Scrum Open が100%で3日連続してから本番へ。

### **4. 受験・申請フェーズ**

- **受験:** 集中できる土日の午前中などがおすすめ。**日本語で受験**すれば翻訳は不要。

- **購入:** [Scrum.org](https://www.scrum.org/assessments/professional-scrum-master-i-certification) からパスワードを購入（$200）

- **実践:** 合格後、jinjerのチーム内で「スクラムガイド的にはこうですよ」と、根拠に基づいた提案を始めます。これが後の「マネジメント経験」の材料になります。
