## ロードマップ（6月〜10月）

### Phase 1：全体像の把握（6月：4週間）

**目的:** 出題範囲を俯瞰し、弱点分野を特定する。

- [ ] テキスト1冊を通読（全部理解しなくてよい。目次と章末問題で「知っている / 知らない」を仕分け）
- [ ] [過去問道場（応用情報）](https://www.fe-siken.com/) で直近1回分を**解説なしで**解き、現在地を把握
- [ ] 計算問題（n進数・確率・応答時間・稼働率）の解法パターンを1冊で整理
- [ ] 午後の選択科目を決定 → **「プロジェクトマネジメント」「サービスマネジメント」** を第一候補（PSM I との相乗効果大）。第三候補は **「経営戦略」**

**週あたり目安:** テキスト 5〜8章 + 道場 100問

---

### Phase 2：午前試験 過去問ドリル（7月〜8月：8週間）

**目的:** 午前の正答率を **80%以上** に安定させる（本番合格ライン60% + 余裕）。

- [ ] [過去問道場](https://www.fe-siken.com/) で **直近10回分** を周回（正答率80%になるまで繰り返し）
- [ ] 間違えた問題は「なぜ外したか」をテキストで確認し、**弱点タグ**（セキュリティ / ネットワーク / DB / アルゴリズム 等）を付ける
- [ ] 計算問題は捨てずに **解法テンプレート** を暗記（下記「計算問題チェックリスト」参照）
- [ ] 8月末：道場の **模擬試験モード** で80問150分を本番形式で1回受験 → **70%以上** を目標

**週あたり目安:** 道場 200〜300問 + 弱点分野のテキスト復習

---

### Phase 3：午後試験 記述対策（9月：4週間）

**目的:** 選択2科目の「回答の型」を体に染み込ませる。

#### 選択科目（推奨）

| 優先 | 科目 | 選ぶ理由 |
| --- | --- | --- |
| ★1 | **プロジェクトマネジメント** | PSM I の知識が直接活きる。WBS・EVM・リスク管理は語彙が近い |
| ★2 | **サービスマネジメント** | ITIL のインシデント / 変更 / 可用性管理。読解中心で得点しやすい |
| ★3 | **経営戦略** | 文章読解力があれば対応可。SWOT・3C・バリューチェーンは用語暗記 |

#### 記述の練習方法

- [ ] **直近5回分** の午後問題を解く（時間を計って150分で2問）
- [ ] 模範解答と自分の答案を **1文ずつ比較** し、「キーワードの拾い方」を分析
- [ ] 答案用紙の **1行25字** 制約に慣れる（行数指定がある設問が多い）
- [ ] 各科目 **「頻出キーワードリスト」** を自作（例：PM → コスト・品質・リスク・WBS・EVM・クリティカルパス）
- [ ] 9月末：午後2問を本番形式で解き、**自己採点60%以上** を確認

**週あたり目安:** 午後過去問 2回分 + 模範解答精読

---

### Phase 4：総仕上げ（10月：試験2週間前まで）

**目的:** 本番当日のパフォーマンスを最大化する。

- [ ] 道場で **直近3回分** を通しで解く（午前80%安定を確認）
- [ ] 午後は **頻出パターンの答案テンプレ** を最終確認（暗記ではなく「型」の確認）
- [ ] 計算問題の **公式カード** を見直し
- [ ] 試験前日：新しい問題は解かない。**間違いノート** とテンプレだけ見る
- [ ] 当日：午前 → 午後の順。午前で60%取れたら撤退判断は不要（部分合格制度あり）

## 計算問題チェックリスト

午前で確実に得点すべき計算パターン（解法を暗記）:

- [ ] **n進数変換**（2進 ↔ 10進 ↔ 16進）
- [ ] **基数変換（小数含む）**
- [ ] **論理演算**（AND / OR / XOR / 否定）
- [ ] **De Morgan の法則**
- [ ] **確率**（反復試行・条件付き確率）
- [ ] **期待値**
- [ ] **応答時間**（ターンアラウンドタイム / スループット）
- [ ] **稼働率・信頼性**（MTBF / MTTR / シリアル・パラレル）
- [ ] **ハミング符号**
- [ ] **木構造の探索**（Preorder / Inorder / Postorder）
- [ ] **ページング / 仮想アドレス** のアドレス計算
- [ ] **通信路の帯域・伝送時間**
