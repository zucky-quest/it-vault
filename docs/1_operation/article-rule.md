# 記事を書く際のルール

## 概要

- 記事を書くプラットフォームの選定について記述

## qiitaとZennは違い

どのように使い分けるのでしょうか？

QiitaとZennは、どちらもエンジニア向けの技術情報共有プラットフォームですが、主な特徴や使い方に違いがあります。

## Qiitaの特徴

- **日本最大級のエンジニア向け情報共有サービス**で、幅広い技術記事が集まっており、初心者から上級者まで多様なユーザーが利用しています。[^1_4][^1_5][^1_1]
- 記事投稿はマークダウン形式で行い、タグやストック機能、いいね、コメントなどコミュニケーションが活発です。
- 検索エンジン評価が高く、広いリーチを持ち、技術的な問題解決目的の記事が多いです。[^1_6]
- 主に**短い技術記事やトラブルシュート、Q\&A**が中心で、収益化の仕組みはほぼありません。


## Zennの特徴

- **深い技術的内容、専門記事や長文にも対応**し、プログラマーが自分の知識を深掘りして発信する傾向があります。
- 記事投稿に加え、**電子書籍（本）や短文メモ（スクラップ）も公開可能**。Markdown対応でGitHub連携もできます。
- Zennは特徴として**有料記事や本の販売、投げ銭で執筆者が収益化できる仕組み**があり、「技術共有＋経済的還元」を目指しています。
- 最新技術（AI、フレームワークなど）に関する深掘り記事や、企業の技術書なども積極的に発信しています。


## 使い分け方

- **Qiita**：幅広いエンジニアに向けて、短め・実践的な技術情報やトラブルシュート・Q\&Aを発信したいときに向いています。多くのエンジニアに記事を読まれたい場合や、カジュアルに知識共有したいときに選ぶと良いです。[^1_1][^1_3]
- **Zenn**：専門的な内容や長文を深掘りして書きたい場合や、記事や電子書籍で**収益化も目指したい**ときに適しています。自分の知見をまとめて出版したい場合や、よりコミュニティ性の強い情報発信を重視したいときにおすすめです。[^1_3][^1_2][^1_7]


## 比較表

| 項目     | Qiita   | Zenn           |
| :----- | :------ | :------------- |
| ユーザー規模 | 非常に多い   | 増加中            |
| 投稿形式   | 記事      | 記事・本・スクラップ     |
| 内容傾向   | 幅広い技術情報 | 専門・深掘り         |
| 収益化    | なし      | あり（有料記事・本・投げ銭） |
| 検索性    | 強い      | 普通〜やや弱い        |

両サービスの違いを理解したうえで、自分の目的や記事の内容、読者層、収益化ニーズに合わせて選択・使い分けるのがおすすめ
